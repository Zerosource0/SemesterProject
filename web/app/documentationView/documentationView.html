Testing


Our test cases were created to both comply with expected values and not. The way we set it up was making two different test classes, one for the rest API and one for our facades used in the rest API. In the rest API test we are testing how to login and extract user specific information. We also tried extracting information when not logged in as any role and expecting to receive a certain status code. For our API we simply tested if the return status 200. We did not check any responses made from the API methods. We tried to use a dedicated persistence unit and database for the testing but we faced issues when doing so. Not only in the tests but it also seemed to cause problems in our application itself. Thus we decided to use the database that the application was using. Yes, it does make our tests less valid but we had to compromise for the sake of functionality.

In the backend testing unlike the API testing we tested more the values that were returned. In the backend testing we have tested whether returns are not empty, making it clear that the method works and returns values and not an error. We mainly tested the adding and deleting of new users as we faced issues in our program when we initially implemented this feature. We then also test the retrieval of currencies which is used in our exchange rate feature. All in all our test cases seem to pass and our system works as wanted, which was the most important thing for us.

Postman Testing

URIs

Our base URI is 'semesterSeedSP/api/'. In order to run the tests you first have to run the web application itself. This will populate tables in your dedicated database and make the tests function. In order to test in postman you will have to add to the base URI. Here is a couple of URIs

Get - Users

/demoadmin/users


Post - Login

/login, you will also have to pass a json object in the body e.g: {\"username\":\"admin\",\"password\":\"test\"}


Get - Exchange Rates

/currency/dailyrates


Post - New User

/newuser, body: {\"username\":\"newUser\",\"password\":\"test\",\"role\":\"admin\"}


Delete - Delete User

/admin/users/{id}


Note: In order to test features that require a login you will have to pass a header in the postman app with the token you receive when logging in.
